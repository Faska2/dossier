Docker & Containers - Guide Complet

1. Qu'est-ce que Docker ?

Docker est une plateforme permettant d'exécuter des applications dans des conteneurs isolés contenant le code, les dépendances et la configuration.

Application → Container → Docker Engine → OS → Machine

Image

Modèle prêt à être exécuté. 

node:20
nginx
mongo
mysql

Container

Instance en cours d'exécution d'une image. 

docker ps
docker ps -a

Docker Hub

Bibliothèque publique d'images Docker. 

docker pull nginx

2. Commandes Docker Essentielles

Version

docker --version

Images

docker images

Containers

docker ps

Télécharger

docker pull nginx

Créer

docker run nginx

Supprimer

docker rm container

3. Gestion des Containers 

docker run -d nginx

docker run -d --name mon-nginx nginx

docker run -d -p 8080:80 nginx

docker stop mon-nginx

docker start mon-nginx

docker restart mon-nginx

docker rm mon-nginx

4. Logs et Debug 

docker logs mon-nginx

docker logs -f mon-nginx

docker exec -it mon-nginx bash

5. Dockerfile 

FROM node:20

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 3000

CMD ["npm","start"]
FROM
Image de base
WORKDIR
Dossier de travail
COPY
Copie des fichiers
RUN
Exécution pendant build
EXPOSE
Port exposé
CMD
Commande de démarrage

6. Build & Run 

docker build -t mon-api .

docker images

docker run -p 3000:3000 mon-api

7. Volumes 

docker volume create data

docker volume ls

docker run -v data:/app/data mon-api

8. Réseaux Docker 

docker network ls

docker network create mon-reseau

docker run --network mon-reseau nginx

9. Docker Compose 

services:

  app:
    build: .
    ports:
      - "3000:3000"

  mongo:
    image: mongo
    ports:
      - "27017:27017"

docker compose up

docker compose up -d

docker compose down

Commandes à retenir pour l'examen

docker images docker ps docker pull docker run docker stop docker start docker logs docker exec docker build docker compose up docker compose down docker rm
