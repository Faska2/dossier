🍃 Qu'est-ce que MySQL ? (Rappel Rapide)

Définition Officielle

MySQL est un Système de Gestion de Base de Données Relationnelle (SGBDR).

  • Langage : SQL (Structured Query Language)
  • Type : Relationnel (tables liées par clés étrangères)
  • ACID garanti : Atomicité, Cohérence, Isolation, Durabilité
  • Open source, très utilisé dans l'industrie

Pourquoi dans les Examens OFPPT ?

  • Fondamental du développement web : Toute application a besoin de données
  • Teste la logique métier : Pas seulement la syntaxe, mais la compréhension du contexte
  • Standard industriel : Utilisé par PHP (Laravel), Python (Django), Node.js...
  • Évaluable objectivement : Syntaxe précise, pas d'ambiguïté

📝 PATTERN #1 : PROCÉDURE STOCKÉE AVEC VALIDATION

TEMPLATE UNIVERSEL - À MÉMORISER PAR CŒUR
DELIMITER // CREATE PROCEDURE NomDeLaProcedure( IN param1 TYPE, IN param2 TYPE, ... ) BEGIN -- ╔════════════════════════════════════════╗ -- ║ ÉTAPE 1 : VÉRIFIER L'EXISTENCE ║ -- ╚════════════════════════════════════════╝ IF NOT EXISTS ( SELECT * FROM TableReference WHERE id_reference = paramReference ) THEN -- Lever une erreur si la contrainte est violée SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Erreur: [Message explicatif]'; END IF; -- ╔════════════════════════════════════════╗ -- ║ ÉTAPE 2 : EFFECTUER L'INSERTION ║ -- ╚════════════════════════════════════════╝ INSERT INTO TableCible ( colonne1, colonne2, id_etranger, ... ) VALUES ( param1, param2, paramReference, ... ); END // DELIMITER ;
⚠️ RÈGLES D'OR À RESPECTER
  1. Toujours commencer par DELIMITER // et finir par DELIMITER ;
  2. Toujours utiliser BEGIN ... END pour encorer le corps
  3. Vérifier l'existence AVANT d'insérer (presque toujours demandé)
  4. Utiliser SIGNAL SQLSTATE '45000' pour lever l'erreur
  5. Les paramètres IN sont les valeurs fournies lors de l'appel

🔢 PATTERN #2 : FONCTION SQL AVEC RETOUR

TEMPLATE UNIVERSEL - À MÉMORISER PAR CŒUR
DELIMITER // CREATE FUNCTION NomDeLaFonction( paramEntree TYPE ) RETURNS TypeDeRetour DETERMINISTIC BEGIN -- ╔════════════════════════════════════════╗ -- ║ DÉCLARER UNE VARIABLE LOCALE ║ -- ╚════════════════════════════════════════╝ DECLARE variableLocale TypeDeRetour; -- ╔════════════════════════════════════════╗ -- ║ EFFECTUER LA REQUÊTE DE CALCUL ║ -- ╚════════════════════════════════════════╝ SELECT fonction_aggregate(champ) OR expression_mathematique INTO variableLocale FROM TableSource WHERE condition; -- ╔════════════════════════════════════════╗ -- ║ RETOURNER LE RÉSULTAT ║ -- ╚════════════════════════════════════════╝ RETURN variableLocale; END // DELIMITER ;
⚠️ DIFFÉRENCES CLÉS AVEC LA PROCÉDURE STOCKÉE
  • RETURNS type : Obligatoire (procédure ne retourne rien)
  • RETURN valeur : Une seule fois à la fin (pas plusieurs RETURN)
  • DECLARE variable : Pour stocker le résultat intermédiaire
  • SELECT ... INTO variable : Pour capturer le résultat d'une requête
  • DETERMINISTIC : Optionnel mais recommandé (même entrée = même sortie)
  • Appel : SELECT NomFonction(param) (pas CALL)

🛡️ PATTERN #3 : TRIGGER DE CONTRÔLE D'INTÉGRITÉ

TEMPLATE UNIVERSEL - À MÉMORISER PAR CŒUR
DELIMITER // CREATE TRIGGER NomDuTrigger -- ╔═════════════════════════════════════════════╗ -- ║ MOMENT : AVANT OU APRÈS L'OPÉRATION ║ -- ╚═════════════════════════════════════════════╝ BEFORE OR AFTER -- ╔═════════════════════════════════════════════╗ -- ║ OPÉRATION DÉCLENCHEUSE ║ -- ╚═════════════════════════════════════════════╝ INSERT OR UPDATE OR DELETE -- ╔═════════════════════════════════════════════╗ -- ║ TABLE CIBLE ║ -- ╚═════════════════════════════════════════════╝ ON NomTableCible -- ╔═════════════════════════════════════════════╗ -- ║ POUR CHAQUE LIGNE AFFECTÉE ║ -- ╚═════════════════════════════════════════════╝ FOR EACH ROW BEGIN -- ╔══════════════════════════════════════════════════╗ -- ║ CAS 1 : BLOQUER L'OPÉRATION (si condition non respectée)║ -- ╚══════════════════════════════════════════════════╝ IF (condition_violee) THEN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Erreur: [Message]'; END IF; -- ╔══════════════════════════════════════════════════╗ -- ║ CAS 2 : MISE À JOUR AUTOMATIQUE (optionnel) ║ -- ╚══════════════════════════════════════════════════╝ -- UPDATE AutreTable SET champ = NEW.valeur WHERE id = NEW.id; END // DELIMITER ;
⚠️ PIÈGE CLASSIQUE : NEW vs OLD

NEW = Valeur APRÈS l'opération

  • Disponible dans INSERT (nouvelle valeur)
  • Disponible dans UPDATE (valeur modifiée)
  • PAS disponible dans DELETE
  • Exemple : NEW.id, NEW.nom

OLD = Valeur AVANT l'opération

  • Disponible dans UPDATE (ancienne valeur)
  • Disponible dans DELETE (valeur supprimée)
  • PAS disponible dans INSERT
  • Exemple : OLD.id, OLD.statut

📋 Scénarios de Trigger Rencontrés

Timing Opération Objectif Exemple
BEFORE INSERT Insertion Empêcher si contrainte non respectée Empêcher utilisation sans voiture affectée
AFTER INSERT Insertion Mettre à jour automatiquement Incrémenter compteur, journaliser
BEFORE UPDATE Modification Valider ou modifier la nouvelle valeur Forcer statut "Retiré" si nb_batteries > 100
BEFORE DELETE Suppression Protéger si dépendances existent Empêcher suppression service avec salariés

CREATE PROCEDURE AfficherNombreSalariesParService()
BEGIN
    SELECT 
        s.nomService,
        COUNT(sa.idSal) AS nombre_salaries

Question 3 4 points

Énoncé : Créer un déclencheur (Trigger) qui empêche, lors de l'insertion d'une nouvelle utilisation, d'affecter une voiture pour un salarié attaché au service 'informatique'.

✅ RÉPONSE CORRECTE
DELIMITER //

CREATE TRIGGER VerifierAffectationVoitureInformatique
    BEFORE INSERT
    ON utilisation
    FOR EACH ROW
BEGIN
    -- Vérifier si le salarié appartient au service 'informatique'
    IF EXISTS (
        SELECT 1 
        FROM salarie sa
        JOIN service s ON sa.codeSer = s.codeSer
        WHERE sa.idSal = NEW.idSal
          AND s.nomService = 'informatique'
    ) THEN
        -- Bloquer l'insertion avec message d'erreur
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'Erreur: Impossible d''affecter une voiture à un salarié du service informatique';
    END IF;
END //

DELIMITER ;

CREATE FUNCTION CoutTotalEvenementsExpert(
    p_prenomExp VARCHAR(50),
    p_nomExp VARCHAR(50)
)
RETURNS DECIMAL(12,2)
DETERMINISTIC
BEGIN
    DECLARE v_coutTotal DECIMAL(12,2);

❌ ERREURS FATALES À NE JAMAIS FAIRE (MySQL)

💀 ERREUR #1 : Oublier DELIMITER

Impact : Erreur de syntaxe immédiate, procédure/fonction/trigger non créée

// ❌ FAUX - Sans DELIMITER
CREATE PROCEDURE MaProc()
BEGIN
    -- Corps
END;
-- Erreur : Le ; à l'intérieur termine l'instruction CREATE prématurément

// ✅ CORRECT - Avec DELIMITER
DELIMITER //
CREATE PROCEDURE MaProc()
BEGIN
    -- Corps
END //
DELIMITER ;
💀 ERREUR #2 : Confondre NEW et OLD dans Trigger

Impact : Logique inversée, corruption de données

// ❌ FAUX - Dans AFTER DELETE (OLD existe, PAS NEW)
CREATE TRIGGER MonTrigger
AFTER DELETE ON table
FOR EACH ROW
BEGIN
    -- Logiquer la suppression avec NEW.id  ← ERREUR !
END;

// ✅ CORRECT - Utiliser OLD pour DELETE/UPDATE avant
CREATE TRIGGER MonTrigger
AFTER DELETE ON table
FOR EACH ROW
BEGIN
    INSERT INTO logs VALUES (OLD.id, OLD.nom);  ← CORRECT
END;
💀 ERREUR #3 : Oublier $set dans UPDATE (équivalent SQL)

Impact : En MySQL, UPDATE sans SET = erreur syntaxe (contrairement à MongoDB)

-- ❌ FAUX - SET manquant
UPDATE Veloid
statut = 'Retiré'
WHERE id = 14;

-- ✅ CORRECT - Avec SET obligatoire
UPDATE Veloid
SET statut = 'Retiré'
WHERE id = 14;
💀 ERREUR #4 : Oublier RETURN dans Function

Impact : Erreur "FUNCTION must return a value"

-- ❌ FAUX - Pas de RETURN
CREATE FUNCTION MaFonction()
RETURNS INT
BEGIN
    DECLARE v INT;
    SELECT COUNT(*) INTO v FROM table;
    -- Oubli de RETURN !
END;

-- ✅ CORRECT - RETURN obligatoire
CREATE FUNCTION MaFonction()
RETURNS INT
BEGIN
    DECLARE v INT;
    SELECT COUNT(*) INTO v FROM table;
    RETURN v;  ← OBLIGATOIRE
END;
💀 ERREUR #5 : Mauvaise gestion des guillemets simples

Impact : Erreur de syntaxe dans les strings

-- ❌ FAUX - Guillemet simple dans string avec apostrophe
SET MESSAGE_TEXT = 'Erreur: L'expert n'existe pas';
                          ^-- L'apostrophe ferme le string !

-- ✅ CORRECT - Échapper l'apostrophe avec \ ou doubler
SET MESSAGE_TEXT = 'Erreur: L''expert n''existe pas';
                                        ^^              ^^
💀 ERREUR #6 : Oublier BEGIN/END

Impact : Seule la première instruction est dans le bloc

-- ❌ FAUX - Sans BEGIN/END
CREATE PROCEDURE MaProc()
    IF condition THEN
        SIGNAL SQLSTATE '45000';  -- Seule instruction exécutée
    END IF;
    INSERT INTO table ...;  -- HORS du bloc IF !

-- ✅ CORRECT - Avec BEGIN/END
CREATE PROCEDURE MaProc()
BEGIN
    IF condition THEN
        SIGNAL SQLSTATE '45000';
    END IF;
    INSERT INTO table ...;  -- Bien dans le même bloc
END

☑️ Avant de Commencer : Les Fondamentaux

  • Je sais que DELIMITER // change le délimiteur de fin d'instruction
  • Je sais que BEGIN ... END encadre un bloc d'instructions
  • Je sais que DECLARE déclare des variables locales
  • Je sais que SELECT ... INTO variable capture un résultat
  • Je sais que SIGNAL SQLSTATE '45000' lève une erreur personnalisée

☑️ Pattern Procédure Stockée (4 pts)

  • Je commence toujours par DELIMITER //
  • J'utilise CREATE PROCEDURE NomProc(IN param TYPE, ...)
  • J'encadre avec BEGIN ... END //
  • Je vérifie l'existence avec IF NOT EXISTS (SELECT ...)
    • SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '...'
  • J'insère avec INSERT INTO ... VALUES (...)
  • Je termine par DELIMITER ;

☑️ Pattern Fonction SQL (4 pts)

  • J'utilise CREATE FUNCTION NomFunc(param TYPE) RETURNS TypeRetour
  • J'ajoute optionnellement DETERMINISTIC
  • Je déclare une variable locale avec DECLARE var TYPE;
  • Je fais ma requête avec SELECT ... INTO var FROM ...
  • Je retourne OBLIGATOIREMENT avec RETURN var;
  • Je n'ai qu'un seul RETURN (à la fin)

☑️ Pattern Trigger (4 pts)

  • Je choisis le bon timing : BEFORE (bloquer/valider) ou AFTER (mettre à jour/journaliser)
  • Je choisis la bonne opération : INSERT, UPDATE, ou DELETE
  • J'ajoute FOR EACH ROW (obligatoire)
  • J'utilise NEW.champ pour la nouvelle valeur (INSERT/UPDATE)
  • J'utilise OLD.champ pour l'ancienne valeur (UPDATE/DELETE)
  • Je vérifie les conditions avec IF ... THEN SIGNAL ... END IF;
  • Je peux faire un UPDATE AutreTable SET ... WHERE ... si nécessaire

  • CREATE ROLE NomRole; pour créer un rôle
  • GRANT SELECT, INSERT, UPDATE, DELETE ON DB.Table TO Role;
  • CREATE USER 'user'@'host' IDENTIFIED BY 'password';
  • GRANT Role TO 'user'@'host'; pour attribuer le rôle
