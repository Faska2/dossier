⚛ Qu'est-ce que React ? (Rappel Rapide)

Définition Officielle

React est une bibliothèque JavaScript open-source pour construire des interfaces utilisateur.

  • Créée par Facebook (2013)
  • Basée sur les composants (réutilisables)
  • Utilise un DOM virtuel (performances)
  • Syntaxe JSX (HTML-like dans JavaScript)
  • Hooks (useState, useEffect) depuis React 16.8

Pourquoi dans les Examens OFPPT ?

  • Standard industriel : Utilisé par Facebook, Netflix, Airbnb...
  • Composants : Architecture modulaire et maintenable
  • React Hooks : Syntaxe moderne demandée
  • Intégration Laravel : API REST facile à consommer
  • Évaluable objectivement : Code clair, pas d'ambiguïté

import { useState } from 'react';

function Composant1() {
  const [voiture, setVoiture] = useState({
    matricule: '',
    marque: '',
    dateCirculation: '',
    couleur: ''
  });

  const [showRecap, setShowRecap] = useState(false);

  const handleChange = (e) => {
    setVoiture({ ...voiture, [e.target.name]: e.target.value });
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    setShowRecap(true);
  };

  return (
    <div>
      <h2>Gestion Voitures</h2>
      
      <form onSubmit={handleSubmit}>
        <input
          type="text"
          name="matricule"
          value={voiture.matricule}
          onChange={handleChange}
          placeholder="Matricule: 1-h-11111"
        />
        
        <select
          name="marque"
          value={voiture.marque}
          onChange={handleChange}
        >
          <option value="">Marque:</option>
          <option value="Toyota">Toyota</option>
          <option value="Renault">Renault</option>
        </select>
        
        <input
          type="date"
          name="dateCirculation"
          value={voiture.dateCirculation}
          onChange={handleChange}
        />
        
        <input
          type="text"
          name="couleur"
          value={voiture.couleur}
          onChange={handleChange}
          placeholder="Couleur: Noir"
        />
        
        <button type="submit">Confirmer</button>
      </form>
      
      {showRecap && (
        <div>
          <h3>Récapitulatif des informations :</h3>
          <ul>
            <li><Matricule:li>
            <li><Marque:li>
            <li><Date Mise en circulation:li>
            <li><Couleur:li>
          </ul>
        </div>
      )}
    </div>
  );
}

❌ ERREURS FATALES À NE JAMAIS FAIRE (React)

💀 ERREUR #1 : Mutation Directe de State

Impact : Le composant ne se re-pas pas, données inchangées à l'écran

// ❌ FAUX - Mutation directe (interdit en React)
const [count, setCount] = useState(0);

function increment() {
    count = count + 1;  ← Ne provoque PAS de re-render !
}

// ✅ CORRECT - Utiliser setter
const [count, setCount] = useState(0);

function increment() {
    setCount(count + 1);  ← Provoque le re-render
}
💀 ERREUR #2 : Oublier key dans .map()

Impact : Warning React "Each child in a list should have a unique key prop", performances dégradées

// ❌ SANS key - Warning + problèmes de performance
{items.map((item) => (
    <div>{item.name}</div>  ← React ne peut pas optimiser
))}

// ✅ AVEC key - Optimisation + identification correcte
{items.map((item) => (
    <div key={item.id}>{item.name}</div>  ← Parfait !
))}
💀 ERREUR #3 : Boucle Infinie avec useEffect

Impact : Boucle infinie d'appels API, crash du navigateur ou consommation mémoire excessive

// ❌ FAUX - Pas de tableau de dépendances = boucle infinie
useEffect(() => {
    fetchData();  ← Exécuté à CHAQUE render !
});

// ✅ CORRECT - Tableau vide = exécution unique au montage
useEffect(() => {
    fetchData();
}, []);  ← Exécuté UNE seule fois
💀 ERREUR #4 : Oublier preventDefault() dans onSubmit

Impact : La page se recharge entièrement, perdant tout le state React

// ❌ FAUX - Page rechargée, state perdu
const handleSubmit = () => {
    setShowRecap(true);  ← Trop tard, le form a déjà été soumis au serveur
};

// ✅ CORRECT - Bloquer le comportement par défaut
const handleSubmit = (e) => {
    e.preventDefault();  ← Bloque le rechargement AVANT tout
    setShowRecap(true);
};
💀 ERREUR #5 : Props Non Déstructurées Correctement

Impact : Accès aux props incorrect, undefined errors

// ❌ FAUX - Accès direct à props (objet)
function Composant(props) {
    return <div>{props.data}</div>;  ← OK mais pas destructuré
}

// ✅ CORRECT - Déstructuring pour clarté
function Composant({ data }) {  ← Extraction explicite
    return <div>{data}</div>;
}
💀 ERREUR #6 : Async/Await mal utilisé dans useEffect

Impact : Promesses non résolues, erreurs non capturées

// ❌ FAUX - Pas de try/catch, promesse non gérée
useEffect(async () => {
    const res = await fetch(url);  ← Si erreur = crash !
    const data = await res.json();
    setData(data);
}, []);

// ✅ CORRECT - Gestion d'erreur complète
useEffect(() => {
    const fetchData = async () => {
        try {
            setLoading(true);
            const res = await fetch(url);
            const data = await res.json();
            setData(data);
        } catch (err) {
            console.error(err);
            setError(err.message);
        } finally {
            setLoading(false);
        }
    };
    fetchData();
}, []);
