☁️ Qu'est-ce que le Cloud Computing ? (Rappel Rapide)

Définition Officielle (NIST)

Le Cloud Computing est un modèle qui permet un accès réseau omniprésent, pratique et à la demande à un pool de ressources informatiques configurables (serveurs, stockage, applications, services) qui peuvent être rapidement provisionnées et libérées avec un minimum d'effort de gestion.

  • 5 caractéristiques essentielles (NIST)
  • 3 modèles de service (IaaS, PaaS, SaaS)
  • 4 modèles de déploiement (Public, Privé, Hybride, Communautaire)

Les 5 Caractéristiques Essentielles (NIST)

  1. Libre-service à la demande : Provisionner des ressources sans interaction humaine
  2. Accès réseau large : Disponible sur tous types de terminaux
  3. Mise en commun des ressources : Mutualisation multi-locataires
  4. Élasticité rapide : Scaling up/down automatique
  5. Service mesuré : Paiement à l'usage (Pay-as-you-go)
💡 Bonne Nouvelle pour Vous

Le Cloud Computing est un thème très structuré avec des définitions précises du NIST. Les questions sont répétitives : connaître les 3 modèles de service, les 4 types de déploiement, et les avantages du cloud vs on-premise.

🏗️ LES 3 MODÈLES DE SERVICE DU CLOUD

🎯 IaaS / PaaS / SaaS = 100% des questions depuis 3 ans

🖥️ IaaS
Infrastructure as a Service

IaaS
  • Fournit : Machines virtuelles, stockage, réseaux
  • Vous gérez : OS, applications, données
  • Exemples : AWS EC2, Google Compute Engine, Azure VM
  • Cas d'usage : Hébergement de sites, sauvegarde, reprise après sinistre
  • Niveau de contrôle : Élevé

⚙️ PaaS
Platform as a Service

PaaS
  • Fournit : Plateforme d'exécution + middleware
  • Vous gérez : Applications et données uniquement
  • Exemples : Heroku, Google App Engine, AWS Elastic Beanstalk
  • Cas d'usage : Déploiement d'applications web, API
  • Niveau de contrôle : Moyen

📦 SaaS
Software as a Service

SaaS
  • Fournit : Application prête à l'emploi
  • Vous gérez : Rien (utilisation uniquement)
  • Exemples : Gmail, Google Drive, Salesforce, Office 365
  • Cas d'usage : Email, CRM, collaboration
  • Niveau de contrôle : Faible

📋 Comparaison Détaillée des 3 Modèles

Caractéristique IaaS PaaS SaaS
Ce que vous gérez Apps + Données + OS + Middleware Apps + Données uniquement Rien (utilisation uniquement)
Ce que le fournisseur gère Virtualisation + Serveurs + Stockage + Réseau OS + Middleware + Runtime + Infra Tout (application + plateforme + infra)
Flexibilité ⭐⭐⭐ Élevée ⭐⭐ Moyenne ⭐ Faible
Coût Pay-as-you-go (ressources) Pay-as-you-go (plateforme) Abonnement par utilisateur
Exemple concret AWS EC2, Azure VM Heroku, Google App Engine Gmail, Office 365

🌐 LES 4 MODÈLES DE DÉPLOIEMENT DU CLOUD

☁️ Cloud Public

Infrastructure partagée accessible au grand public via Internet.

  • Propriétaire : Fournisseur tiers (AWS, Azure, GCP)
  • Avantage : Faible coût, élasticité maximale
  • Inconvénient : Moins de contrôle, sécurité partagée

🏢 Cloud Privé

Infrastructure dédiée à une seule organisation.

  • Propriétaire : L'organisation elle-même
  • Avantage : Contrôle total, sécurité renforcée
  • Inconvénient : Coût élevé, maintenance interne

🔀 Cloud Hybride

Combinaison de cloud public et privé avec orchestration.

  • Fonctionnement : Données sensibles sur privé, calcul sur public
  • Avantage : Flexibilité, optimisation des coûts
  • Inconvénient : Complexité de gestion

👥 Cloud Communautaire

Infrastructure partagée entre plusieurs organisations ayant des préoccupations communes.

  • Exemple : Cloud gouvernemental, cloud santé
  • Avantage : Coût partagé, conformité réglementaire
  • Inconvénient : Gouvernance complexe

🖥️ VIRTUALISATION & HYPERVISEUR

🖥️ La virtualisation est le fondement technologique du Cloud

La virtualisation permet d'exécuter plusieurs systèmes d'exploitation (invités) sur un seul serveur physique (hôte) en utilisant un hyperviseur.

🏗️ Types d'Hyperviseurs

Type Description Exemples
Type 1 (Bare Metal) S'exécute directement sur le matériel physique sans OS hôte. Performances proches du natif. VMware ESXi, Microsoft Hyper-V, KVM, Xen
Type 2 (Hébergé) S'exécute sur un OS hôte. Adapté pour les postes de travail et les tests. VirtualBox, VMware Workstation, Parallels

✅ Avantages de la Virtualisation

  • 📉 Réduction des coûts matériels (consolidation de serveurs)
  • ⚡ Provisionnement rapide (minutes vs jours)
  • 🔒 Isolation des environnements
  • 🔄 Snapshots et migration à chaud
  • 🌍 Meilleure utilisation des ressources (ratio ~10:1)

📊 Virtualisation vs Cloud

  • Virtualisation : Abstraction du matériel, une VM = un OS
  • Cloud : Virtualisation + orchestration + self-service + billing
  • Le Cloud utilise la virtualisation comme technologie de base
  • On peut avoir de la virtualisation sans Cloud (on-premise)
  • On ne peut pas avoir de Cloud sans virtualisation

🐳 DOCKER & CONTENEURISATION

🐳 Docker : La conteneurisation moderne

Un conteneur est une unité standardisée de logiciel qui empaquette le code et toutes ses dépendances pour que l'application s'exécute rapidement et de manière fiable d'un environnement à un autre.

📋 Conteneur vs Machine Virtuelle

Caractéristique Conteneur (Docker) Machine Virtuelle
OS Partage le kernel de l'hôte OS invité complet (kernel + librairies)
Taille Mégaoctets (MB) Gigaoctets (GB)
Démarrage Secondes Minutes
Isolation Niveau processus (légère) Niveau matériel (forte)
Performance Natif (pas d'overhead) Léger overhead (~5-10%)
Portabilité Excellente (Docker Engine partout) Bonne (compatible hyperviseur)

🐳 Commandes Docker Essentielles

Commandes Docker Fondamentales

✅ AVANTAGES & INCONVÉNIENTS DU CLOUD

✅ Avantages du Cloud Computing
  • Économies d'échelle : Pas d'investissement initial (CAPEX → OPEX)
  • Élasticité : Scaling automatique selon la demande
  • Haute disponibilité : SLA 99.9% à 99.999%
  • Paiement à l'usage : On paie uniquement ce qu'on consomme
  • Mobilité : Accès depuis n'importe où, n'importe quel appareil
  • Mise à jour automatique : Le fournisseur gère la maintenance
  • Reprise après sinistre : Backup et DRP intégrés
⚠️ Inconvénients & Risques
  • Dépendance au fournisseur : Vendor lock-in
  • Sécurité des données : Données hébergées chez un tiers
  • Conformité légale : RGPD, lois sur la souveraineté des données
  • Latence réseau : Dépend de la qualité de la connexion Internet
  • Coûts cachés : Transferts de données sortants, bande passante
  • Disponibilité : Dépend du fournisseur (panne AWS/Azure)

📝 EXAMEN JUIN 2024 - VARIANTE V1

Juin 2024 - Variante 1
10 Points | Cloud Computing
Question 1 3 points

Énoncé : Expliquer le modèle "Responsabilité Partagée" (Shared Responsibility Model) dans le Cloud.

✅ RÉPONSE CORRECTE

Le modèle de responsabilité partagée définit ce que le fournisseur cloud sécurise et ce que le client doit sécuriser.

Le fournisseur (AWS/Azure/GCP) est responsable DE LA sécurité DU Cloud :

  • • Sécurité physique des datacenters
  • • Infrastructure réseau et hyperviseur
  • • Matériel et stockage

Le client est responsable DE LA sécurité DANS le Cloud :

  • • Configuration des pare-feux et groupes de sécurité
  • • Gestion des accès et identités (IAM)
  • • Chiffrement des données côté client
  • • Sécurité des applications déployées

En IaaS : Plus de responsabilités pour le client. En SaaS : Plus de responsabilités pour le fournisseur.

Question 2 4 points

Énoncé : Qu'est-ce que Docker ? Expliquer le concept d'image et de conteneur.

✅ RÉPONSE CORRECTE

Docker est une plateforme de conteneurisation qui permet d'empaqueter, distribuer et exécuter des applications dans des environnements isolés appelés conteneurs.

📦 Image Docker : Modèle en lecture seule (template) contenant le code, les dépendances, les librairies et la configuration nécessaires pour exécuter une application. Une image est construite à partir d'un Dockerfile.

🚀 Conteneur Docker : Instance exécutable d'une image Docker. Un conteneur est léger, isolé, et peut être démarré/arrêté en quelques secondes.

Analogie : L'image Docker est comme une classe (POO) et le conteneur est comme une instance de cette classe.

10 Points | Cloud Computing
Question 1 3 points

Énoncé : Définir le "Vendor Lock-in" dans le contexte du Cloud. Comment l'éviter ?

✅ RÉPONSE CORRECTE

Le Vendor Lock-in (dépendance au fournisseur) est la situation où un client devient dépendant d'un fournisseur cloud au point qu'il est difficile et coûteux de migrer vers un autre fournisseur.

Causes :

  • Utilisation de services propriétaires (ex : AWS DynamoDB, Azure Cosmos DB)
  • Formats de données spécifiques
  • Coûts de transfert de données sortantes élevés

Comment l'éviter :

  • Utiliser des technologies open source (Kubernetes, Terraform, Docker)
  • Architecture multi-cloud ou hybride
  • Standardiser avec des APIs REST et des formats de données ouverts
Question 2 4 points

Énoncé : Expliquer le concept d'élasticité dans le Cloud. Donner un exemple concret avec AWS/Azure.

✅ RÉPONSE CORRECTE

L'élasticité est la capacité d'un système cloud à augmenter ou diminuer automatiquement les ressources allouées en fonction de la charge de travail.

Types d'élasticité :

  • Scale Up (vertical) : Augmenter la puissance d'une machine (plus de RAM/CPU)
  • Scale Out (horizontal) : Ajouter plus d'instances derrière un load balancer

Exemple concret :

  • AWS Auto Scaling : Configure un groupe d'instances EC2 qui augmente automatiquement pendant les heures de pointe et diminue la nuit
  • Cas typique : Site e-commerce Black Friday → Le nombre de serveurs passe de 5 à 50 automatiquement

Question 1 4 points

Énoncé : Qu'est-ce que le Serverless Computing ? Donner 2 avantages et 2 inconvénients.

✅ RÉPONSE CORRECTE

Le Serverless Computing (ou Fonction-as-a-Service) est un modèle d'exécution cloud où le fournisseur gère dynamiquement l'allocation des ressources. Le développeur écrit et déploie du code sans se soucier de l'infrastructure sous-jacente.

Exemples : AWS Lambda, Azure Functions, Google Cloud Functions

✅ Avantages

  • Pas de gestion de serveur
  • Paiement à l'exécution (pas de coût fixe)
  • Scalabilité automatique infinie

⚠️ Inconvénients

  • Cold start (latence au premier appel)
  • Timeout (max 15 min sur AWS Lambda)
  • Difficile à déboguer localement
Question 2 4 points

Énoncé : Expliquer le concept de Container Orchestration. Quel est le rôle de Kubernetes ?

✅ RÉPONSE CORRECTE

L'orchestration de conteneurs est la gestion automatisée du déploiement, de la mise à l'échelle, de la mise en réseau et de la disponibilité des conteneurs.

Kubernetes (K8s) est la plateforme d'orchestration de conteneurs la plus populaire. Ses fonctionnalités incluent :

  • Déploiement automatisé : Rollout et rollback des applications
  • Auto-scaling : Ajuste le nombre de pods selon la charge
  • Service Discovery : Les conteneurs se découvrent automatiquement
  • Load Balancing : Distribue le trafic entre les pods
  • Auto-réparation : Redémarre les conteneurs défaillants

Question 1 4 points

Énoncé : Qu'est-ce que le Cloud Act et son impact sur la souveraineté des données ?

✅ RÉPONSE CORRECTE

Le Cloud Act (Clarifying Lawful Overseas Use of Data Act, 2018) est une loi américaine qui permet aux autorités américaines d'accéder aux données stockées par les entreprises américaines, où qu'elles se trouvent dans le monde.

Impact sur la souveraineté des données :

  • ⚔️ Conflit avec le RGPD : Le RGPD européen interdit le transfert de données personnelles hors UE sans garanties
  • 🌍 Les entreprises européennes doivent vérifier où leurs données sont stockées
  • 🔒 Solutions : Chiffrement côté client, cloud souverain (OVH, Deutsche Telekom)
  • 📋 Le concept de data residency (résidence des données) devient crucial

❌ ERREURS FRÉQUENTES EN CLOUD COMPUTING

💀 ERREUR #1 : Confondre IaaS et PaaS
❌ FAUX : "Dans le PaaS, je gère aussi l'OS et le middleware."

✅ CORRECT : "En PaaS, le fournisseur gère l'OS, le middleware et le runtime. Je gère uniquement mes applications et données."
💀 ERREUR #2 : Virtualisation = Cloud
❌ FAUX : "La virtualisation et le Cloud Computing sont la même chose."

✅ CORRECT : "La virtualisation est une technologie qui permet d'exécuter plusieurs OS sur un serveur. Le Cloud utilise la virtualisation + orchestration + self-service + paiement à l'usage."
💀 ERREUR #3 : Cloud = Pas de sécurité
❌ FAUX : "Le Cloud n'est pas sécurisé, mes données sont exposées."

✅ CORRECT : "La sécurité est un modèle de responsabilité partagée. Le fournisseur sécurise l'infrastructure, le client doit configurer correctement ses services (IAM, chiffrement, pare-feu)."
💀 ERREUR #4 : Conteneur = VM
❌ FAUX : "Un conteneur Docker est une machine virtuelle légère."

✅ CORRECT : "Un conteneur partage le kernel de l'hôte alors qu'une VM a son propre OS. Le conteneur est un processus isolé, pas une virtualisation matérielle."
