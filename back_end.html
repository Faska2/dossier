🍃 Qu'est-ce que Laravel ? (Rappel Rapide)

Définition Officielle

Laravel est un framework PHP open-source pour le développement d'applications web suivant le pattern MVC (Modèle-Vue-Contrôleur).

  • Modèle = Eloquent ORM (Base de données)
  • Vue = Blade Templates (Interface utilisateur)
  • Contrôleur = Logique métier (Traitement des requêtes)
  • Routes = URL → Contrôleur@Méthode

Pourquoi Laravel dans les Examens OFPPT ?

  • Standard industriel : Le framework PHP le plus populaire au monde
  • Eloquent ORM puissant : Relations faciles (hasMany, belongsTo, belongsToMany)
  • Blade Templates : Moteur de templates élégant et sécurisé
  • Artisan CLI : Commandes pour générer du code (make:model, make:controller...)
  • Migration : Versioning de la base de données

📦 PATTERN #1 : MIGRATION (Création de Table)

TEMPLATE UNIVERSEL - Création de Table
// Commande Artisan pour générer le fichier : php artisan make:migration create_nom_table_table // Fichier généré : database/migrations/xxxx_xx_xx_xxxxxx_create_nom_table_table.php use Illuminate\Database\Migrations\Migration; use Illuminate\Database\Schema\Blueprint; use Illuminate\Support\Facades\Schema; class CreateNomTableTable extends Migration { /** * Run the migrations. * * @return void */ public function up() { Schema::create('nom_table', function (Blueprint $table) { $table->id(); $table->string('colonne1'); $table->string('colonne2'); $table->foreignId('autre_id')->constrained(); $table->timestamps(); }); } /** * Reverse the migrations. * * @return void */ public function down() { Schema::dropIfExists('nom_table'); } };
⚠️ Points Clés à Retenir
  • Schema::create('nom_table') : Crée la table
  • $table->id() : Clé primaire auto-incrémentée (INT UNSIGNED AUTO_INCREMENT)
  • $table->string('champ') : VARCHAR(255) par défaut
  • $table->foreignId('autre_id')->constrained() : Clé étrangère avec contrainte
  • $table->timestamps() : created_at ET updated_at automatiques
  • down() : Toujours implémenter (rollback possible)
  • Exécution : php artisan migrate

🗂️ PATTERN #2 : MODÈLE ELOQUENT AVEC RELATIONS

TEMPLATE UNIVERSEL - Modèle + Relations
// Commande Artisan pour générer le modèle : php artisan make:model NomModele // Fichier généré : app/Models/NomModele.php namespace App\Models; use Illuminate\Database\Eloquent\Model; class NomModele extends Model { /** * The attributes that are mass assignable. * * @var array */ protected $fillable = [ 'colonne1', 'colonne2', 'id_etranger', ]; /** * Relation One-to-Many (ce modèle possède plusieurs enfants) */ public function enfants() { return $this->hasMany(Enfant::class, 'id_parent'); } /** * Relation Many-to-One (ce modèle appartient à un parent) */ public function parent() { return $this->belongsTo(Parent::class, 'id_etranger'); } /** * Relation Many-to-Many (avec table pivot) */ public function tags() { return $this->belongsToMany(Tag::class, 'pivot_table'); } }
⚠️ Les 3 Types de Relations à Maîtriser

hasMany() → Un-à-Plusieurs

Exemple : Un Service a plusieurs Salariés

// Dans Service.php
public function salaries() {
    return $this->hasMany(Salarie::class, 'codeSer');
}

belongsTo() → Plusieurs-à-Un

Exemple : Un Salarié appartient à un Service

// Dans Salarie.php
public function service() {
    return $this->belongsTo(Service::class, 'codeSer');
}

belongsToMany() → Plusieurs-à-Plusieurs

Exemple : Un Événement a plusieurs Participants (et vice-versa)

// Dans Evenement.php
public function participants() {
    return $this->belongsToMany(
        Participant::class,
        'inscriptions',  // table pivot
        'evenement_id', // FK dans table pivot
        'participant_id'  // FK dans table pivot
    );
}

$fillable ⚠️ OBLIGATOIRE

Sans $fillabe, erreur "MassAssignmentException"

protected $fillable = [
    'nom',
    'email',
    'telephone'
];

TEMPLATE UNIVERSEL - Resource Controller Complet
// Commande Artisan pour générer le contrôleur : php artisan make:controller NomController --resource // Fichier généré : app/Http/Controllers/NomController.php namespace App\Http\Controllers; use App\Models\NomModele; use Illuminate\Http\Request; class NomController extends Controller { /** * Display a listing of the resource. */ public function index() { // Récupérer tous les enregistrements $donnees = Modele::with(['relation1', 'relation2']) ->get(); // Optionnel : Filtrer si paramètre présent if (request->has('filtre')) { $donnees = $donnees->where('champ', request->input('filtre')); } // Retourner la vue avec les données return view('nomvue.index', ['donnees' => $donnees]); } /** * Show the form for creating a new resource. */ public function create() { // Afficher formulaire de création return view('nomvue.create'); } /** * Store a newly created resource in storage. */ public function store(Request $request) { // Valider les données $validated = $request->validate([ 'champ1' => 'required', 'champ2' => 'required|email', ]); // Créer l'enregistrement Modele::create($validated); Rediriger vers liste avec message succès return redirect()->route('nomroute.index') ->with('success', 'Élément créé avec succès'); } /** * Display the specified resource. */ public function show($id) { // Trouver l'enregistrement avec ses relations $item = Modele::with(['relation1', 'relation2']) ->find($id); return view('nomvue.show', ['item' => $item]); } /** * Show the form for editing the specified resource. */ public function edit($id) { $item = Modele::find($id); return view('nomvue.edit', ['item' => $item]); } /** * Update the specified resource in storage. */ public function update(Request $request, $id) { $validated = $request->validate([...]); $item = Modele::find($id); $item->update($validated); return redirect()->route('nomroute.index') ->with('success', 'Élément modifié avec succès'); } /** * Remove the specified resource from storage. */ public function destroy($id) { $item = Modele::find($id); $item->delete(); return redirect()->route('nomroute.index') ->with('success', 'Élément supprimé avec succès'); ⚠️ Erreurs Fréquentes dans les Contrôleurs
  • Oublier with(['relation']) → N+1 queries problème (lazy loading)
  • Oublier validate() → Données invalides en BDD
  • Oublier $fillable dans le modèle → MassAssignmentException
  • Mauvais nom de route → Erreur 404
  • Oublier redirect()->with('success', ...) → Pas de feedback utilisateur

⚡ Directives Blade à Connaître Absolument

Structure de Base

  • @extends('layout.app') : Hériter le layout principal
  • @yield('content') : Définir la section principale
  • @section('content') ... @endsection : Contenu de la section

Affichage & Boucles

  • {{ $variable }} : Affichage échappé (auto-escaped)
  • {!! $html !!} : Affichage brut (HTML non échappé)
  • @foreach ($array as $item) ... @endforeach : Boucle
  • @if(condition) ... @endif : Condition
  • @empty($array) ... @endempty : Si tableau vide

Formulaires & Sécurité

  • @csrf : Token CSRF obligatoire (POST forms)
  • @method('DELETE') : Méthode HTTP masquée (PUT/PATCH/DELETE)
  • {{ route('nom.route', $id)}} : Génération d'URL
  • {{ session('key') }} : Messages flash

    {
        $voitures = Voiture::whereHas('utilisations', function ($query) {

            'voitures' => $voitures,
            'codeSal' => $codeSal
        ]);
    }
}

Q3 (11pts) : Contrôleur MaterielController

Énoncé : Créer avec les méthodes :

  1. index.blade.php() : qui permet de retourner et d'afficher la liste matériels dans la vue
  2. AfficherMateriels() : qui permet de retourner et d'afficher la liste matériels
  3. AjouterMateriel() : qui permet d'ajouter une nouveau matériel rediriger vers l'action
  4. SupprimerMateriel() : qui permet de supprimer un matériel dont le code est passé en paramètre puis rediriger vers l'action AfficherListeMateriels()
✅ RÉPONSE
class MaterielController extends Controller
{
    public function index()
    {
        $materiels = Materiel::with('affectations')->get();
        return view('materiel.index', ['materiels' => $materiels]);
    }

    public function AfficherListeMateriels()
    {
        $materiels = Materiel::all();
        return view('materiel.liste', ['materiels' => $materiels]);
    }

    public function AjouterMateriel(Request $request)
    {
        $validated = $request->validate([
            'codeMat' => 'required|unique:materiels',
            'marque' => 'required',
            'categorie' => 'required',
        ]);

        Materiel::create($validated);

        return redirect()->route('materiel.AfficherListeMateriels')
                     ->with('success', 'Matériel ajouté avec succès');
    }

    public function SupprimerMateriel($codeMat)
    {
        $materiel = Materiel::find($codeMat);
        $materiel->delete();

        return redirect()->route('materiel.AfficherListeMateriels')
                     ->with('success', 'Matériel supprimé avec succès');
    }
}

☑️ Pattern Migration (2pts)

  • J'écris toujours Schema::create('nom_table', function($table))
  • J'ajoute $table->id() pour clé primaire auto-incrémentée
  • J'utilise $table->foreignId('fk')->constrained() pour clés étrangères
  • J'ajoute $table->timestamps() pour created_at/updated_at
  • J'implémente toujours down() avec Schema::dropIfExists()

☑️ Pattern Modèle (4-5pts)

  • protected $fillable = [...] → OBLIGATOIRE (MassAssignmentException sinon)
  • hasMany(Relation::class, 'fk') → Un-à-Plusieurs
  • belongsTo(Parent::class, 'fk') → Plusieurs-à-Un
  • belongsToMany(Relation::class, 'pivot_table') → Plusieurs-à-Plusieurs
  • Je connais les 3 types de relations par cœur

☑️ Pattern Contrôleur (9-11pts) ⭐ COEUR DU DOSSIER

  • index() → Model::with(['relations'])->get() + view('vue.index')
  • show($id) → Model::with([...])->find($id) + view('vue.show')
  • store(Request $request) → $request->validate([...]) + Model::create($validated) + redirect
  • update(Request $request, $id) → Validation + Model::find($id)->update($validated)
  • destroy($id) → Model::find($id)->delete() + redirect avec message
  • Je sais utiliser with() pour éviter N+1 problem
  • Je sais gérer les messages flash session('success', '...')

☑️ Pattern Vue Blade (4-6pts)

  • @extends('layouts.app') → Hériter le layout principal
  • @yield('content') ou @section('content')...@endsection → Définir section
  • @foreach ($items as $item) ... @endforeach → Boucle
  • {{ $variable }} → Affichage échappé
  • {!! $html !!} → Affichage HTML brut
  • @csrf → Token CSRF (obligatoire dans POST)
  • @method('DELETE') → Masquer méthode DELETE/PUT
  • {{ route('nom.route', $id) }} → Génération URL
  • @if(session('success'))... → Afficher message flash

☑️ Pattern Route (1-3pts)

  • Route::resource('url', Controller::class) → 7 routes automatiques
  • ->name('nom') dans définition → Pour utiliser route('nom')
  • Je connais la structure : GET/POST/PUT/DELETE → index/store/update/destroy
