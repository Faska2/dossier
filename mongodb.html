🍃 Qu'est-ce que MongoDB ? (Rappel Rapide)

Définition Officielle

MongoDB est une base de données NoSQL (Not Only SQL) de type document-oriented.

  • Stocke les données au format BSON (Binary JSON)
  • Pas de schéma fixe (schema-less)
  • Structure hiérarchique flexible
  • Évite les jointures complexes (embedded documents)

Pourquoi dans les Examens OFPPT ?

  • Stack moderne Full Stack : MEAN/MERN stack utilise MongoDB
  • Alternative à MySQL : Montre que vous connaissez les deux paradigmes
  • Facile à évaluer : Syntaxe simple, questions standardisées
  • Industry standard : Très utilisé dans le monde réel

⚡ LES 6 COMMANDES MONGODB ESSENTIELLES (À MAÎTRISER PAR CŒUR)

🎯 Ces 6 commandes = 100% des questions MongoDB depuis 3 ans

Mémorisez-les parfaitement et vous n'aurez aucune surprise le jour J.

1. Créer une Base de Données et une Collection
// Sélectionner/Créer la base de données
use nomBaseDeDonnees;

// Créer une collection (équivalent d'une table)
db.createCollection("nomCollection");
Exemple concret :

// Créer la base "DBSalaries"
use DBSalaries;

// Créer la collection "salaries"
db.createCollection("salaries");
💡 Points clés :
  • use crée la base si elle n'existe pas (lazy creation)
  • La base est créée réellement seulement quand on insère le premier document
  • createCollection() permet de définir des options (validation, capped, etc.)
2. Insérer des Documents (INSERT)
// Insérer UN seul document
db.collection.insertOne({ ... });

// Insérer PLUSIEURS documents
db.collection.insertMany([ { ... }, { ... }, ... ]);
Exemple typique d'examen (documents fournis) :

db.salaries.insertMany([
  {
    "_id": "s1",
    "nomSal": "Alami",
    "prenomSal": "Sara",
    "fonction": "Technicien",
    "service": { "codeser": "I", "nomser": "Informatique" }
  },
  {
    "_id": "s2",
    "nomSal": "Hilal",
    "prenomSal": "Hamza",
    "fonction": "Gérant",
    "service": { "codeser": "L", "nomser": "Logistique" }
  }
]);
💡 Points clés :
  • L'énoncé fournit toujours les documents JSON à insérer
  • Copiez-les simplement en respectant la syntaxe
  • _id peut être automatique (ObjectId) ou manuel (string)
  • Les documents peuvent contenir des sous-documents (objets imbriqués)
3. Lire/Trouver des Documents (FIND) avec Tri (SORT)
// Trouver TOUS les documents
db.collection.find();

// Trouver avec condition (filtre)
db.collection.find({ champ: valeur });

// Trier les résultats
db.collection.find().sort({ champ: 1 }); // 1 = croissant, -1 = décroissant
Question type examen : "Afficher les salariés par ordre croissant des noms"

db.salaries.find().sort({ "nomSal": 1 });
💡 Points clés :
  • "Ordre croissant" = { champ: 1 }
  • "Ordre décroissant" = { champ: -1 }
  • On peut combiner find() + sort() en chaîne
  • Le tri se fait toujours après le find()
4. Compter les Documents (COUNT)
// Compter TOUS les documents
db.collection.countDocuments();

// Compter avec condition (filtre)
db.collection.countDocuments({ condition });
Exemples typiques d'examens :

// Compter les salariés ayant la fonction "Technicien"
db.salaries.countDocuments({ "fonction": "Technicien" });

// Compter les événements ayant le thème "Développement Web"
db.evenements.countDocuments({ "theme": "Développement Web" });

// Compter les batteries dont le statut est "en panne"
db.batteries.countDocuments({ "statut": "en panne" });
💡 Points clés :
  • Retourne un nombre entier (integer), pas un curseur
  • La condition utilise la même syntaxe que find()
  • Très souvent utilisée avec une condition d'égalité simple
5. Mettre à Jour des Documents (UPDATE)
// Mettre à jour UN seul document (le premier trouvé)
db.collection.updateOne(
  { /* filtre */ },
  { $set: { champ: nouvelleValeur }
);

// Mettre à jour TOUS les documents correspondants
db.collection.updateMany(
  { /* filtre */ },
  { $set: { champ: nouvelleValeur }
);
Exemples réels d'examens :

// Modifier le coût journalier de l'événement _id "e2" à 600
db.evenements.updateOne(
  { "_id": "e2" },
  { $set: { "cout_journalier": 600 }
);

// Modifier santé_batterie à "faible" pour toutes les batteries capacité ≤ 50%
db.batteries.updateMany(
  { "capacité": { $lte: 50 },
  { $set: { "sante_batterie": "faible" }
);

// Modifier la valeur du coût journalier du séminaire _id "s2" à 700
db.seminaires.updateOne(
  { "_id": "s2" },
  { $set: { "cout_journalier": 700 }
);
💡 Points CRITIQUES :
  • OUBLIER $set = ERREUR FATALE ! Sans $set, le document est remplacé entièrement
  • $set modifie seulement les champs spécifiés
  • updateOne() = premier document trouvé uniquement
  • updateMany() = tous les documents correspondants
  • Opérateurs de comparaison : $lt (moins que), $lte (≤), $gt (plus que), $gte (≥)
6. Supprimer des Documents (DELETE)
// Supprimer UN seul document (le premier trouvé)
db.collection.deleteOne({ condition });

// Supprimer TOUS les documents correspondants
db.collection.deleteMany({ condition });
Exemples typiques d'examens :

// Supprimer le salarié ayant l'_id "s3"
db.salaries.deleteOne({ "_id": "s3" });

// Supprimer tous les événements ayant durée > 15 jours
db.evenements.deleteMany({ "durée": { $gt: 15 } });

// Supprimer toutes les batteries statut "en panne" ET capacité < 20%
db.batteries.deleteMany({
  "statut": "en panne",
  "capacité": { $lt: 20 }
});
💡 Points clés :
  • deleteOne() supprime le premier document qui matche
  • deleteMany() supprime tous les documents qui matchent
  • La condition peut combiner plusieurs critères (AND implicite)
  • Retourne un objet avec deletedCount (nombre de suppressions)

📊 Différences Fondamentales

MySQL (Relationnel)

  • Schéma fixe : Définir colonnes/types avant
  • Normalisation : Éviter redondance
  • Jointures : Nécessaires pour relier tables
  • Transactions : ACID garanti
  • Langage : SQL (déclaratif)

MongoDB (Document)

  • Schéma flexible : Documents différents OK
  • Dénormalisation : Données dupliquées autorisées
  • Embedding : Sous-documents intégrés
  • Eventual consistency : BASE
  • Langage : Méthodes chaînées (impératif)

    "_id": "2",

    "numero_serie": "345",

Question 3 2 points

Énoncé : Écrire une requête qui Modifie le champ sante_battery à "faible" pour toutes les batteries dont le statut est en panne et la capacité est inférieure ou égale à 50%.

✅ RÉPONSE (Attention : Condition composée !)
db.batteries.updateMany(
  {
    "statut": "en panne",
    "capacity": { $lte: "50%" }
  },
  {
    $set: { "sante_battery": "faible" }
  }
);

// Note: $lte = less than or equal (inférieur ou égal)
// Ici capacity est stocké comme string "50%", donc comparaison lexicographique
// Si c'était un nombre: { $lte: 50 }
Question 4 1 point

Énoncé : Afficher les attributs numero_cycles, sante_battery et capacity des batteries dont le nombre de cycle est entre 600 et 1000 classes en ordre décroissant.

✅ RÉPONSE
db.batteries.find(
  {
    "nombre_cycles": { $gte: 600, $lte: 1000 }
  },
  {
    "numero_cycles": 1,
    "sante_battery": 1,
    "capacity": 1
  }
).sort({ "nombre_cycles": -1 });

// $gte = greater than or equal (supérieur ou égal)
// $lte = less than or equal (inférieur ou égal)
// -1 = ordre décroissant
Question 5 1 point

Énoncé : Afficher le nombre de batteries regroupées par nombre de cycles en statut.

✅ RÉPONSE (Agrégation avancée)
db.batteries.aggregate([
  {
    $group: {
      "_id": {
        "nombre_cycles": "$nombre_cycles",
        "statut": "$statut"
      },
      "nombre": { $sum: 1 }
    }
  }
]);

// Regroupement multi-champs (compound grouping)

Question 4 2 points

Énoncé : Afficher les attributs nombre_cycles, sante_battery et capacity des batteries dont la capacité est différente de 50% des batteries par nombre de cycles en ordre décroissant.

✅ RÉPONSE
db.batteries.find(
  {
    "capacity": { $ne: "50%" }
  },
  {
    "nombre_cycles": 1,
    "sante_battery": 1,
    "capacity": 1
  }
).sort({ "nombre_cycles": -1 });

$ne = not equal (différent de)
Question 5 2 points

Énoncé : Afficher le nombre de batteries regroupées par nombre de cycles.

✅ RÉPONSE
db.batteries.aggregate([
  {
    $group: {
      "_id": "$nombre_cycles",
      "nombre": { $sum: 1 }
    }
  }
]);

✅ RÉPONSE (Opérateur $nin)
db.batteries.deleteMany({
  "statut": { $nin: ["en service", "en retenue"] }

$nin = not in (pas dans la liste)

❌ ERREURS FATALES À NE JAMAIS FAIRE

💀 ERREUR #1 : Oublier $set dans UPDATE

Impact : Le document entier est remplacé au lieu d'être mis à jour !

// ❌ FAUX - Remplace tout le document !
db.salaries.updateOne(
  {_id: "s1"},
  {"fonction": "Ingénieur"}  // Sans $set = remplacement total
);

// ✅ CORRECT - Modifie seulement le champ
db.salaries.updateOne(
  {_id: "s1"},
  {$set: {"fonction": "Ingénieur"}}  // Avec $set = mise à jour ciblée
);
💀 ERREUR #2 : Confondre deleteOne vs deleteMany

Impact : Suppression de trop ou pas assez de documents

// ❌ FAUX si on veut supprimer TOUS les "Technicien"
db.salaries.deleteOne({"fonction": "Technicien"});  // Supprime seulement 1 !

// ✅ CORRECT pour supprimer tous
db.salaries.deleteMany({"fonction": "Technicien"});  // Supprime tous

// ❌ FAUX si on veut supprimer 1 document précis
db.salaries.deleteMany({_id: "s1"});  // Supprime tous avec _id=s1 (normalement 1 seul)

// ✅ CORRECT pour supprimer 1 seul
db.salaries.deleteOne({_id: "s1"});  // Supprime le premier trouvé
💀 ERREUR #3 : Mauvais Opérateur de Comparaison

Impact : Résultats incorrects ou vides

// ❌ FAUX - Utilise = au lieu de $gt/$lt
db.batteries.find({"capacity": "> 50%"});

// ✅ CORRECT - Utilise les opérateurs MongoDB
db.batteries.find({"capacity": {$gt: "50%"}});

// ❌ FAUX - Confondre $lt et $lte
db.evenements.deleteMany({"duree": {$lt: 15}});  // Supprime durée < 15

// ✅ CORRECT - Lire attentivement l'énoncé
db.evenements.deleteMany({"duree": {$gt: 15}});  // Supprime durée > 15
💀 ERREUR #4 : Oublier les Guillemets pour les Strings

Impact : Erreur de syntaxe ou résultat vide

// ❌ FAUX - Valeur string sans guillemets
db.salaries.countDocuments({fonction: Technicien});

// ✅ CORRECT - Toujours mettre des guillemets
db.salaries.countDocuments({"fonction": "Technicien"});

📝 Tableau des Opérateurs de Comparaison (À Mémoriser)

Opérateur Signification Exemple Équivalent SQL
$eq Égal (equal) {"age": {$eq: 25}} WHERE age = 25
$ne Différent (not equal) {"status": {$ne: "active"}} WHERE status != 'active'
$gt Plus grand que (greater than) {"price": {$gt: 100}} WHERE price > 100
$gte Plus grand ou égal {"age": {$gte: 18}} WHERE age >= 18
$lt Plus petit que (less than) {"qty": {$lt: 20}} WHERE qty < 20
$lte Plus petit ou égal {"score": {$lte: 50}} WHERE score <= 50
$in Dans la liste {"status": {$in: ["A", "B"]}} WHERE status IN ('A','B')
$nin Pas dans la liste (not in) {"type": {$nin: ["food"]}} WHERE type NOT IN ('food')
